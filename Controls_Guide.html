Controls — User Guide

Overview

The Controls page is used on the day of the rally to record marshal and control times, calculate penalties, and publish results. The Controls page reads the list and order of controls from the Schedule page and displays a corresponding set of control cards for each rally day. Controls do not create or change the schedule,

Page details, blue font represents none editable data, only clerk of the course (COC) penalties and ATA times can be edited.
Results can be either entered manually or results can be automatically populated from Cloud data – refer Control data.

Key items

·      A card is shown for each control defined in the Schedule, STC’s, Marshal’s, Open Control’s, and Start STC’s, each numbered as per schedule sequence.

·      Controls are processed per day using the day selector.

·      Expected times (ETA) are calculated automatically.

·      Actual times (ATA) are entered manually or automatically.

·      Scores are calculated from the difference between ATA and ETA, max errors 600.

·      Save and Load use the same workspace as Admin and Schedule.

·      Print results- presentation options and a summary for multiply rally days.

·      Export, creates an Excel file of the result which you can then tailor.

 

Page layout

Purpose:
To provide a clear, card-based layout for entering control times and viewing calculated results.

Description:

On the left of the page is the listing of entrant details, their speed group, Rally number and lastly their total error score, these field cannot be edited. The entrant Rally number when clicked will open a card enabling the results for that entrant to be entered sequentially rather than panning to individual control cards across the page.

To the right of the entrant details is an optional card for the entry of COC penalties, the “offence” and the points to be awarded, to include these penalties in the final score COC must be checked. Control Cards are displayed left to right in the order they appear in the Schedule. Each card shows one row per Rally number (entrant).

 

Day selection

Purpose:
To select which rally day’s controls are being viewed and edited, when you select an alternate day the schedule for that day will be automatically selected, likewise the same applies selecting a day in schedule, the controls for that day will be presented.

How to use:

·      Use the Day selector at the top of the page.

·      Select the required day.

·      The visible cards and data update immediately.

Control cards

Purpose:
Each card represents one control point and is used to calculate times and scores for each of the entrants based on rally number and speed group.

Card types:

·      STC — Self time control cards as defined in the Schedule

·      Marshal — records arrival times at a fixed control point.

·      Open Control — similar to Marshal but without certain restrictions.

·      Start STC — resets the timing baseline.

 

STC, Marshal and Open Control cards

Purpose:
To record arrival times (ATA) and calculate penalties, penalties ate limited to +/- 600 seconds, the total score is the absolute sum of penalties.

Expected times (ETA) are calculated and cannot be edited. Blue font.

·      ETA time calculations.

§  STC - elapsed time from Schedule based on SG of entrant.

§  Marshals and Open Controls – entrant start time + elapsed time from Schedule based on the SG of entrant.

·      If a Start STC precedes the control: ETA = Start STC time + elapsed time from Schedule.

Note: If a Start STC time has not been entered, then following card ETA times will remain blank.

How to use:

·      Review the calculated ETA.

·      Enter the ATA as recorded by the entrants and marshals.

·      The score updates automatically.


Start STC cards

Purpose:
A Start STC resets the rally timing. All following ETA times are calculated from this point. STC and Marshal ETA times being calculated based on the Start STC time submitted.

How to use:

·      Enter the Start STC time provided by each entrant.

·      Once entered, downstream ETAs update automatically.

·      Until a Start STC time is entered, expected times for later controls cannot be calculated.

·      Start STC cards do not record penalties themselves.

 

Note: when a Start STC is incorporated into the schedule, later marshal timings are no longer predicable, so this needs to be appreciated and arrangements made accordingly.

Result Audit

Purpose:
A tool that will highlight result errors that exceed the audit value selected, to use the tool first select it (check) and then enter a value. Should a control have excessive errors which could be a result of an unexpected route delay, then that control can be removed from the overall results by unchecking it. All control cards are normally checked. In print the results from an unchecked control will be zero.

Typical workflow

·      Build the route and controls in Schedule.

·      Open Controls on rally day.

·      Select the correct day.

·      Enter ATA times provided by each entrant

·      Enter ATA Marshal times.

·      Review scores or make use of the Audit tool

·      Deselect any controls to be excluded from the day’s result

·      Print results

Controls data (Google STC & Marshal ATA imports)

Purpose:
STC and Marshal times can be submitted electronically, once submitted the results can be downloaded, the control card ATA times populated ready for review and printings of the day results, thus reducing or eliminating manual entries on the rally day.

How it works:
When Controls data is set up and connected, the page can fetch the latest control records from the cloud and populate ATA fields for the relevant controls and entrants.

How to use:
Click the Controls data button, a card will be presented, if the cloud service has been setup in Admin then “Connected” will be displayed at the top of the card.

 

Card buttons

·      View response forms – opens the Google responses page.

·      Open control data – opens the rally worksheet, select the required tab, review the data awaiting download.

·      Download controls data – only enabled when there is data available for download, select to import data populating the control cards.

Note: Each download will overwrite previous data.

·      Upload control data – used when all entries and schedule plannings have been finalized.

·      Entrant and Marshal links, either copy, email or WhatsApp (WA) links to the results forms to be completed.

·      To WA a link you will need to have WA on your computer, you then send the link to yourself, which you can then share as required.

Note: Consider setting up a WA group of entrants, Marshal forms will need to be sent separately, or marshal group.

Notes:

Entrant results are checked against their Surname and Rally number, if an error is made that submitted will not be added to the worksheet data tab ready for download, it will be included in the error list tab.

Entrant and Marshals can submit more than one return, the last being used.

When a download is performed it will overwrite any previously captured data.
